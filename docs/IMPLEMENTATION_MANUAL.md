> 结合本项目 VIS 合规约束、三大答辩场景、静态 HTML 技术栈，

## 一、设计思想篇：三篇文章教给我们什么

在动手改代码之前，先理解这三篇文章的**思维框架**。它们讲的不是"按钮放哪里"，而是"做设计决策时脑子应该怎么转"。

### 1.1 陈西：把复杂问题拆成简单零件

陈西文章的核心方法论只有一个：**分解**。

> PPT 封面 = 标题文案 + 装饰元素

这不是一句废话。他是在说：面对一个复杂的封面设计任务，不要凭感觉一把抓，而是把它拆成两个独立问题——

- **标题文案**：决定风格方向。科技型标题 → 科技风模板；发布会型标题 → 大气风模板。标题文案一旦确定，整套模板的视觉基调就定了。
- **装饰元素**：实现视觉丰富度。全图型（大面积图片）、半图型（图片+蒙版）、无图型（纯色块线条）。三者不是互斥的，可以组合使用，只是比重不同。

同样的分解思维贯穿全文：

| 页面类型 | 分解方式 |
|----------|---------|
| 封面 | 标题文案 + 装饰元素（全图/半图/无图） |
| 目录 | 常规型 / 创意型（创意点靠装饰元素） |
| 内页 | 图文混排 / 纯图片 / 纯文字 / 逻辑图表 / 数据图表 |
| 结束页 | 延用封面设计，首尾呼应 |

他的另一个关键洞察：**不知道怎么做？去看。** 打开一个模板网站，只看封面，忽略所有文案，只看剩余的装饰元素——它们的层级关系、排版、位置关系、元素选取。这是一种训练设计眼光的方法。

**对本项目的启示**：
- 我们已经有 9 套模板，但每套的封面装饰元素偏少（只有 2-3 种 CSS 装饰），需要丰富
- 当前的封面全部是"无图型"——缺少全图型和半图型的变体
- 内页类型不够丰富——缺少纯图片展示页、逻辑图表页
- "去看"的方法：可以找一个优秀的 PPT 模板网站（如 slidesgo、envato 上的学术/竞选模板），分析它们的装饰元素用法

### 1.2 稿定设计：从目的倒推设计

稿定文章的核心方法论是：**目的决定设计**。

它给出了一个决策链条：

```
演示目的是什么？
  → 观众是谁？
    → 他们需要什么信息？
      → 用什么样的结构和视觉来传达？
```

具体的 8 条原则可以归纳为三个层面：

**策略层**（动手前想清楚）：
- 明确演示目的（汇报成果？争取选票？展示学习？）
- 分析受众特征（教授评委？同学投票人？任课老师？）

**结构层**（内容组织）：
- 划分章节（引言→主体→结论）
- 简明扼要（一页一个核心观点，避免信息过载）

**执行层**（视觉呈现）：
- 字体一致性（1-2 种字体，标题/正文分明）
- 选对图表类型（柱状图比大小，折线图看趋势，饼图看占比）
- 动画适度（突出重点而非炫技，淡入淡出比旋转飞入更实用）
- 反复检查（拼写、语法、外部审阅）

**对本项目的启示**：
- 学术/课程/竞选三种场景的**目的和受众完全不同**，模板设计应该体现这种差异
- 当前模板中，三种场景的区分主要在颜色和页面上，但在**内容组织逻辑**上差异不够大
- 学术答辩的观众（教授）需要看到严谨的论证链条；竞选演讲的观众（同学）需要看到可信的个人形象
- 每页应该只承载一个核心观点——如果一张 slide 放了 3 段文字 + 2 个图表，信息就过载了

### 1.3 JENNY：先有骨架，再长血肉

JENNY 文章的核心方法论是：**内容先行，逻辑为骨**。

她区分了两个容易混淆的概念：

> "模板设计"——只有框架没有内容的 PPT，用户拿回去填自己的内容
> "项目制作"——针对具体某次演讲制作的实际幻灯片

**本项目做的是"模板设计"**。这意味着：
- 我们提供的不是成品幻灯片，而是**内容容器**
- 用户拿到模板后，替换文字和图片就能用
- 模板的质量取决于"容器"的通用性和适配性

JENNY 提出的内容梳理三步法：

```
第一步：明确逻辑关系（先后顺序？总分结构？并列对比？）
第二步：用思维导图确定内容框架
第三步：提炼文字 → 关键词 → 短句 → 图形化
```

然后是**导航系统**——JENNY 认为这是 PPT 的骨架：

> 封面 → 目录 → 章节切换页 → 正文页（每页带标题栏） → 结尾页

导航系统的作用是确保观众**始终知道三件事**：
1. 整场演讲有几个部分（目录给的全局认知）
2. 当前讲到哪个部分（过渡页 + 章节指示器给的定位）
3. 接下来还要讲多久（进度指示给的预期）

**对本项目的启示**：
- 当前模板的导航系统只有页码——缺少章节指示器和进度指示
- 缺少过渡页（章节切换页），观众无法感知"换场"
- 模板的占位文字应该用场景化中文而非 lorem ipsum，因为购买者需要通过占位文字**想象自己内容填进去的样子**
- "阅览→收集→模仿→思考→原创"的审美训练路径，对设计模板的人同样适用

### 1.4 三篇文章的交集：一个决策框架

把三篇文章的思想合并，得到以下决策框架：

```
         ┌─────────────────────────────┐
         │  为什么要做这个演示？（稿定）   │  ← 目的决定一切
         │  观众是谁？他们关心什么？       │
         └─────────────┬───────────────┘
                       ↓
         ┌─────────────────────────────┐
         │  内容逻辑是什么？（JENNY）     │  ← 思维导图 → 章节划分
         │  先后？总分？并列？对比？       │
         └─────────────┬───────────────┘
                       ↓
         ┌─────────────────────────────┐
         │  用什么结构承载？（三篇一致）   │  ← 封面→目录→过渡→内页→结束
         │  23 页公式：1+1+4×4+1        │
         └─────────────┬───────────────┘
                       ↓
         ┌─────────────────────────────┐
         │  用什么视觉呈现？（陈西+稿定）  │  ← 标题文案决定风格
         │  装饰元素 + 动画 + 图表        │     装饰元素决定质感
         └─────────────────────────────┘
```

这个框架将指导后续所有设计决策。

## 二、场景分析篇：理解三种答辩的本质差异

三篇文章都强调"目的和受众决定设计"。本项目恰好有三个截然不同的场景。

### 2.1 学术答辩：证明研究的严谨性

| 维度 | 特征 |
|------|------|
| **目的** | 让评委相信：这个问题重要、方法正确、结果可靠 |
| **观众** | 教授、专家（专业、挑剔、关注细节、时间有限） |
| **核心信息** | "我做了一个有价值的研究，且有数据支撑" |
| **情感基调** | 严谨、专业、谦逊、自信但不张扬 |
| **时间** | 通常 15-20 分钟，评委可能在中途走神 |

**内容逻辑应该是**（对应 JENNY 的思维导图步骤）：

```
问题是什么？（背景）
  → 别人怎么做的？（文献）
    → 我做了什么不同的事？（方法）
      → 效果如何？（实验+数据）
        → 有什么不足？（局限+展望）
```

这是典型的**递进逻辑**——每一步都是下一步的前提。观众必须跟上这个链条，少了一环就会困惑。

**当前模板的问题**：
- `kind-background` 只有 1 页，但实际上"研究背景"通常需要：背景概述 + 文献综述 + 研究动机 + 关键问题，至少 4 页
- `kind-framework` 和 `kind-data` 之间缺少"研究方法"的详细展开
- 缺少"对比分析"页（你的方法 vs 别人的方法，数据对比）
- 缺少"局限与展望"页（评委必问）

**模板应该提供什么**：
- 清晰的章节递进结构（4 个过渡页分别对应 4 个论证阶段）
- 数据展示的多种方式（指标卡、柱状图、折线图、表格）
- 逻辑关系图（总分、递进、对比）
- 引用和注释的排版方式（学术 PPT 经常需要标注数据来源）

### 2.2 课程展示：证明学习的过程和成果

| 维度 | 特征 |
|------|------|
| **目的** | 展示团队做了什么、学到了什么、成果如何 |
| **观众** | 任课老师 + 同班同学（友善但也在比较） |
| **核心信息** | "我们发现了一个真实问题，设计了解决方案，并且验证了它" |
| **情感基调** | 协作、活力、设计感、年轻 |
| **时间** | 通常 10-15 分钟，多个小组轮换 |

**内容逻辑应该是**：

```
发现了什么问题？（用户调研）
  → 谁有这个痛点？（用户画像）
    → 我们设计了什么方案？（方案+原型）
      → 用户怎么说？（测试反馈）
        → 我们怎么分工的？（团队+时间线）
```

这是典型的**设计思维逻辑**——以用户为中心，从问题到方案到验证。

**当前模板的问题**：
- 缺少"用户调研数据"的展示方式（问卷统计、访谈引用）
- 缺少"原型对比"页（改进前 vs 改进后）
- 缺少"团队协作流程"页（不只列出人名，而是展示协作关系）
- 卡片风格的内页变体不够多

**模板应该提供什么**：
- 用户画像卡片（多种排版变体）
- 原型展示布局（图片+标注+说明）
- 团队组织图/协作流程图
- 对比展示（before/after）

### 2.3 竞选答辩：证明个人的能力和承诺

| 维度 | 特征 |
|------|------|
| **目的** | 让投票人相信：我有能力、我有计划、我值得信任 |
| **观众** | 同学、老师（感性+理性判断，投票决定结果） |
| **核心信息** | "我做过实事、我看到了问题、我有可行的方案" |
| **情感基调** | 自信、真诚、有力量感、可亲近 |
| **时间** | 通常 5-10 分钟，多人竞选，需要快速建立印象 |

**内容逻辑应该是**：

```
我是谁？（个人介绍）
  → 我做过什么？（成果展示）
    → 我看到了什么问题？（痛点分析）
      → 我准备怎么做？（工作计划）
        → 我承诺什么？（宣言+行动表）
```

这是典型的**信任建立逻辑**——过去的行为（成果）是未来行为（承诺）的最好证明。

**当前模板的问题**：
- `kind-profile` 只有 1 页，个人介绍太单薄——竞选答辩中这是最关键的"第一印象"页
- `kind-achievements` 是时间线形式，但竞选成果更适合用"数字+图片"展示
- 缺少 SWOT 分析页（评优评先答辩经常需要）
- 缺少"他人评价/推荐"的展示方式
- 结束页应该强化"请投我一票"的行动号召

**模板应该提供什么**：
- 个人介绍的多种排版（照片+文字、时间线、能力雷达图）
- 成果展示的"数字墙"（大数字+简短说明）
- SWOT 分析四象限布局
- 引用卡片（他人评价、推荐语）
- 百日行动时间线（具体到周/月）

### 2.4 场景差异对照表

| 设计维度 | 学术答辩 | 课程展示 | 竞选答辩 |
|----------|---------|---------|---------|
| 核心逻辑 | 递进论证 | 设计思维 | 信任建立 |
| 节奏 | 稳重、从容 | 明快、有节奏 | 紧凑、有力 |
| 数据角色 | 核心证据 | 辅助说明 | 成果佐证 |
| 图片角色 | 方法/结果配图 | 原型/场景展示 | 个人/活动照片 |
| 动画风格 | 克制（线扫、淡入） | 活泼（弹入、缩放） | 庄重（金色扫光、按压） |
| 封面装饰 | 科技网格、光效 | 圆角色块、彩带 | 红绸、金光、光束 |
| 过渡页比重 | 中（强调章节逻辑） | 中（强调设计阶段） | 低（快速推进） |
| 结束页重点 | 致谢+联系方式 | 致谢+Q&A | 行动号召+投票 |

这个对照表是后续所有设计决策的基础。每个模板的每个页面，都应该服务于对应场景的核心逻辑。

## 三、结构设计篇：从 10 页到 24 页

### 3.1 为什么是 24 页？

陈西给出了 PPT 模板的黄金公式：

```
1 封面 + 1 目录 + (1 过渡页 + 4 内页) × 4 章节 + 1 结束页 = 23 页
```

这个公式不是拍脑袋的——它对应着一场标准答辩的内容承载需求：

- **每章节 4 页内页**：保证购买者修改时有足够的排版变体可选。如果只有 1 页面，用户只能"填文字"，没有排版选择的余地。
- **4 个章节**：对应大多数答辩汇报的逻辑结构（引言→方法→结果→结论）
- **过渡页**：给观众"换场"的心理信号，是 JENNY 所说的导航系统的关键节点

对于本项目，微调为 **24 页**（学术/课程）和 **25 页**（竞选，因为多了"承诺宣言"环节）。

### 3.2 学术模板：递进论证结构（24 页）

遵循 JENNY 的"思维导图先行"原则，先画出学术答辩的内容框架，再映射到页面结构：

```
论证主线：
  问题有意义 → 方法有创新 → 实验有说服力 → 结论有贡献

章节设计：
  第一章：为什么做这个研究？（建立问题重要性）
  第二章：怎么做的？（展示方法创新性）
  第三章：效果如何？（用数据证明）
  第四章：有什么意义和不足？（诚实而有信心地收束）
```

```
06  background-3         研究动机                       引用数据+指标卡，量化问题重要性
```


### 3.4 竞选模板：信任建立结构（25 页）

## 四、视觉语言篇：让模板"长对样子"

陈西说"标题文案决定风格"，在本项目中，这个原则更进一步：**场景决定了视觉语言**。

### 4.1 学术模板的视觉语言

**核心原则**：装饰服务于论证。所有视觉元素的存在，是为了让数据更清晰、让逻辑更可见，而不是为了好看。

**封面**（陈西的"标题文案+装饰元素"）：

当前封面（`templates/academic-tech-dark/index.html:376-404`）的装饰元素：
```html
<div class="ornament-ring"></div>     <!-- 环形光效 -->
<div class="ornament-ribbon"></div>   <!-- 缎带装饰 -->
<div class="ornament-badge"></div>    <!-- 徽章 -->
<div class="research-grid"></div>     <!-- 科研网格 -->
```

这些是"无图型"封面（陈西分类）。建议增加**半图型**变体：
- 右侧 40% 放校园建筑照片 + 深色蒙版
- 左侧 60% 放标题文案
- 装饰元素简化为金色细线（少即是多）

**图表区**：当前 `generate.cjs` 中学术模板的数据展示使用 CSS 柱状图（`.bars` + `.bar`）。这是好的——纯 CSS 不依赖 JS。但需要增加变体：
- 水平条形图（适合对比多个方法的长名称）
- 表格样式（适合展示数据集信息、超参数）
- 公式/算法伪代码区域（学术答辩高频需求）

**动画**：学术场景的动画应该"安静"——线扫（lineSweep）、淡入（fadeUp）是主体。避免弹跳（bounce）和旋转——在严肃的学术场合显得轻浮。

### 4.2 课程模板的视觉语言

**核心原则**：明亮、友好、有设计感。课程展示的观众是老师和同学，不需要学术答辩的严肃感。

**封面**：当前课程模板（`src/templates/templates.js:171-216`）使用暖纸底色 + 圆角色块 + 蓝橙配色。方向是对的。需要增强：
- 增加半图型封面变体（校园场景 + 彩色蒙版）
- 装饰元素增加更多"圆"和"曲线"（和学术的"直线+网格"形成对比）

**卡片系统**：课程模板大量使用卡片（用户画像、功能列表、团队成员）。当前 `generate.cjs` 的 `persona` 和 `solution` case 已经有三栏卡片布局，但没有充分利用 CSS：
- 卡片应该有微妙阴影（`box-shadow`）营造"浮起来"的感觉
- 不同卡片可以用不同的彩色顶部边框（主色/辅色/金色）
- 悬停效果（虽然是静态 HTML，但可以做视觉层次）

**动画**：可以比学术模板更活泼——scaleIn（弹入）、stagger（逐条跳入）是合适的。避免过度使用。

### 4.3 竞选模板的视觉语言

**核心原则**：庄重、有力量、可信。红色在中国文化中代表权威和庆典，金色代表荣誉和品质。

**封面**：当前竞选模板使用庆典红 + 金色 + 团徽章印 + 红绸。方向是对的。但红绸的 CSS 实现比较简单：
```css
/* 当前的 ribbon-gold 是简单的金色分割线 */
.ribbon-gold { /* ... */ }
```

建议增强：
- 增加光束效果（`conic-gradient` 模拟舞台聚光灯）
- 红绸使用更复杂的渐变 + 曲线路径（SVG 实现）
- 封面增加半图型变体（演讲者照片 + 红色蒙版）

**"数字墙"布局**：竞选模板的成果展示页应该用"大数字 + 简短标签"——
```
┌──────────┐  ┌──────────┐  ┌──────────┐
│   12     │  │  3000+   │  │   95%    │
│ 场大型活动 │  │ 服务人次  │  │ 满意度   │
└──────────┘  └──────────┘  └──────────┘
```
数字要足够大（48px+），标签要简洁（不超过 5 个字）。这是竞选答辩中最常见也最有冲击力的成果展示方式。

**动画**：金色扫光（goldSweep）是竞选模板的独有动画——目前只有这个场景使用。按压弹出（badgeStamp）适合团徽。避免使用过于轻浮的弹跳动画。

### 4.4 导航系统的统一视觉

JENNY 强调的导航系统，在三种场景中应该有统一的逻辑但不同的视觉呈现：

**学术模板**：
```
brand-header:  [校徽+校名]  ● ③ 实验分析  ACADEMIC DEFENSE  14/24
slide-footer:  ● ③ 实验分析  ●●●○○  哈尔滨工业大学（深圳）  14
```
指示器用青色圆点（匹配 `--accent: #45d6c8`），进度点用细线连接。

**课程模板**：
```
brand-header:  [校徽+校名]  ◆ ② 方案设计  COURSE PROJECT  10/24
slide-footer:  ◆ ② 方案设计  ◆◆◇◇◇  哈尔滨工业大学（深圳）  10
```
指示器用橙色菱形（匹配 `--accent: #ff7a3d`），更活泼。

**竞选模板**：
```
brand-header:  [校徽+校名]  ★ ③ 问题诊断  CAMPAIGN DEFENSE  14/25
slide-footer:  ★ ③ 问题诊断  ★★★☆☆  哈尔滨工业大学（深圳）  14
```
指示器用金色星形（匹配 `--gold: #f5c66b`），呼应仪式感。

**过渡页的编号设计**（三种场景各不同）：

```css
/* 学术：描边数字，科技感 */
.transition-number {
  font-size: 160px;
  font-weight: 900;
  color: transparent;
  -webkit-text-stroke: 2px rgba(69,214,200,.15);
}

/* 课程：半透明填充，柔和 */
.course .transition-number {
  font-size: 140px;
  color: rgba(0,83,117,.06);
}

/* 竞选：金色填充，力量感 */
.campaign .transition-number {
  font-size: 160px;
  color: rgba(245,198,107,.12);
  text-shadow: 0 0 40px rgba(245,198,107,.08);
}
```

## 附录：三篇文章核心语录

> **陈西**："封面页的组成主要分为两个大的版块：第一、标题文案。第二、装饰元素。所有的PPT封面页的设计都是由这两个版块组成。"

> **陈西**："每一个过渡页下面最好搭配4个或以上的内页。"

> **陈西**："PPT模板的结束页的设计都是延用封面页的设计。这样可以形成首尾呼应的效果。"

> **稿定设计**："在开始制作PPT之前，要明确演示的目的。是要向观众介绍一个新产品、传达某项数据、还是分享某个主题的见解？不同的演示目的需要不同的设计和内容策略。"
